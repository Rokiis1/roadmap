# Content of developer tools (DevTools) level 1

**Browser Developer Tools**, commonly called **DevTools**, are built-in **inspection** and **diagnostic** tools available in web browsers such as **Chrome**, **Edge**, **Firefox**, and **Safari**. They allow **observation**, **analysis**, and **inspection** of how a web application is structured, styled, and executed inside the browser.

**DevTools** provide visibility into three fundamental aspects of a web application **structure**, **appearance**, and **behavior**.

The **structural aspect** is represented through the document model of the page. This allows inspection of how **content is organized into elements** such as **headings**, **paragraphs**, **forms**, **buttons**, and **containers**.

The **visual aspect** is represented through **styling rules**. DevTools expose font **properties**, **colors**, **spacing**, **alignment**, and **layout positioning**. These properties define how information is presented to users and how consistently it appears across different devices and screen sizes.

The **behavioral aspect** is represented through **scripts** and **runtime execution**. DevTools display **messages**, **warnings**, and **errors** generated during page execution. These messages reveal whether scripts executed correctly or failed, and whether application logic behaved as expected.

DevTools also expose **network activity**. This includes requests sent to **servers**, **responses received**, **loading times**, and **resource sizes**. Network visibility allows observation of how data is transferred and how long different parts of the application take to load.

Another important capability of DevTools is **environment simulation**. Screen size, resolution, device type, and network conditions can be simulated to observe how applications behave under different technical conditions.

DevTools further provide access to **storage mechanisms** such as **cookies**, **local storage**, and **session storage**. These areas reveal how applications store and retrieve data on the client side.

At **DevTools Level 1**, are understood as an **observation** and **inspection** environment rather than a development environment. The objective is not to modify code, but to understand how web applications are constructed and executed inside the browser.

DevTools therefore represent a bridge between visible application behavior and invisible technical processes. They make hidden system behavior observable and measurable.
