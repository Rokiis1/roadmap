## Syntax Level 1

### Summary Syntax Level 1

In this level, we learned the **fundamental syntax rules** that define how Python code is written and structured.

We began by exploring **indentation and code blocks**, which form the backbone of Python’s structure. We learned that indentation replaces braces and determines how statements, loops, and functions are grouped and executed.

Next, we covered **comments**, learning how to document code using single-line comments and multi-line docstrings. Comments allow developers to explain intent without affecting program execution.

We then introduced **variables**, understanding them as **names that reference objects** rather than containers that store values directly. We learned naming conventions such as `snake_case` and the use of uppercase names for configuration-style constants.

After that, we worked with **literals and basic types**, using fixed values such as numbers, strings, and booleans to assign data to variables. This established how values first enter a program through syntax.

We then answered a key conceptual question: **what is an object?**  
We learned that everything in Python is an object, and variables simply point to these objects in memory.

Finally, we explored **built-in functions**, such as `print()`, `type()`, `len()`, `dir()`, and `help()`. These functions allow us to interact with objects, inspect them, and understand how Python programs operate internally.

Together, Syntax Level 1 provides the **essential grammar of Python**, forming the foundation needed to read, write, and understand basic programs.

## Syntax Level 2

### Summary Syntax Level 2

In this level, we expanded our understanding of syntax by focusing on **how objects are created and reused**, and by introducing the **syntax for defining new object types**.

We began by revisiting **objects**, moving beyond the idea that “everything is an object” to understanding **where objects come from**. We saw that objects can be created through **literals**, **function calls**, and **class instantiation**, and that variables continue to act only as references to these objects.

Next, we introduced **class syntax**. We learned that a class is a **blueprint** for creating objects and that defining a class does not create an object by itself. Instead, it defines the structure that future objects of that type will follow.

Finally, we learned how to **create objects from classes**. By calling a class like a function, Python creates a new object known as an **instance**. Even when created from the same class, each instance is a separate object with its own identity.

At this stage, classes are introduced **only as syntax**, without behavior or internal logic. The design and behavior of objects are explored later in **Object-Oriented Programming Level 1**.

Together, Syntax Level 2 acts as a **bridge between basic syntax and object-oriented thinking**, preparing the learner to understand how Python programs define and work with custom objects.
