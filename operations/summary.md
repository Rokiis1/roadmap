## Operations Level 1

### Summary Operations Level 1

In this level, we learned the fundamentals of **Python operations**, focusing on how values are manipulated using operators.

We began by introducing **operators and operands**, learning the difference between **unary operators**, which act on a single value, and **binary operators**, which combine two values. This established the foundation for understanding how expressions are formed and evaluated.

Next, we explored **assignment and augmented assignment operators**, learning how values are stored in variables and how shorthand operators such as `+=`, `-=`, and `*=` simplify updating existing values. We also discussed common errors such as **division by zero** and **type mismatches** when combining incompatible data types.

Finally, we studied the **order of operations for arithmetic expressions**. We learned that Python follows rules similar to mathematics, evaluating parentheses first, then exponentiation, followed by multiplication and division, and finally addition and subtraction. We also saw how **parentheses** can be used to explicitly control evaluation order, which is especially important when translating math and physics formulas into code.

## Operations Level 2

### Summary Operations Level 2

In this level, we extended our understanding of operations beyond arithmetic and focused on how Python evaluates **conditions, logic, and object relationships**.

We began by exploring **comparison operators**, which allow values to be compared to determine relationships such as equality, inequality, and ordering. We also examined important edge cases, including **floating-point comparison behavior** and **type compatibility requirements**.

Next, we introduced **logical operators and truthiness**, learning how Python evaluates conditions using `and`, `or`, and `not`. We saw how non-boolean values can participate in logical expressions based on whether they are considered **truthy** or **falsy**, enabling more expressive conditional logic.

We then examined **membership operators**, which allow us to test whether a value exists within a container such as a list, string, or dictionary, and learned how membership checks behave differently depending on the container type.

After that, we covered **identity operators**, which determine whether two variables refer to the same object in memory rather than merely containing the same values. This distinction is especially important when working with mutable objects.

Finally, we brought all these concepts together by studying **operator precedence and associativity**. We learned how Python determines the evaluation order when arithmetic, comparison, logical, membership, and identity operators are combined in a single expression, and why understanding this order is essential for writing correct and readable code.
